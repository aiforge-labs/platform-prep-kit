Design Patterns

Architectural patterns used throughout the system

🔄
ReAct Reasoning
Observe → Reason → Act → Evaluate loop in every agent
🤝
Multi-Agent
5 specialized agents coordinated by orchestrator
🔁
Write-Through Sync
SQLite ↔ Markdown bidirectional state sync
🛡️
CSRF Protection
Signed tokens on every mutating request
💾
Session Resume
Quiz sessions persisted to SQLite, resume on return
🔎
Fuzzy Matching
Topic → knowledge pack lookup via keyword scoring
🔌
Provider Agnostic
Bridge pattern decouples from any AI provider
📐
Adaptive Plans
Low quiz scores insert review days, high scores compress

Data Storage

Dual storage with automatic sync — CLI uses Markdown, Web UI uses SQLite

📝
~/.prep/tracker.md
CLI source of truth. Day-by-day progress as Markdown tables. Human-editable.
🗄️
~/.prep/prep.db
Web UI source of truth. SQLite with WAL mode for concurrent access. 8 tables.
⚙️
~/.prep/config.yml
User config: target role, timeline, AI provider, reminders. YAML format.
📊
~/.prep/quiz-history.json
Append-only quiz results with scores, weak areas, timestamps.
📅
~/.prep/study-plan.md
Generated plan overview. Read-only reference (tracker.md is authoritative).
🔒
Privacy
All data local. No telemetry. No cloud. Unencrypted by design (personal tool).
